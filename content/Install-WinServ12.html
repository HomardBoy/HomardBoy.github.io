Procédure d’installation d’une machine Windows Server 2012 (R2)


I - Prerequis.

• L’installation d’un Serveur Windows 2012 doit se préparer dans un premier temps au niveau hardware. Deux options s’offrent à nous, à savoir une installation sur machine virtuelle ou sur machine dédiée. Dans les deux cas, il faut respecter la configuration d’au moins 35 Go d’espace disque libre et de 4 Go de mémoire vive. En dessous d’une tel configuration, l'expérience de configuration des services risque d’être difficile et la machine risque d’avoir un temps de réaction trop long. Un ISO du système d’exploitation est également nécessaire. Pour une VM, la simple présence du fichier sur la machine est obligatoire. Pour un serveur physiquement dédié, l’ISO doit être brûlé sur une clé USB bootable ou sur un CD d’installation. La possession d’une clé d’activation correspondant à la version de l’OS est également nécessaire. Dans le cas suivant, la configuration se fera sur une machine virtuelle Vmware Workstation Pro 12, mais le processus reste identique peut importe le logiciel de virtualisation ou la machine dont il est question.

I - Installation.

• La première des étapes consiste à créer une nouvelle machine virtuelle et à indiquer l’emplacement du fichier ISO. La suite de la configuration est classique, à savoir indiquer le nom de la machine, l’emplacement des fichiers sur la machine hôte, l’espace disque, le mode d’attribution de l’espace disque et les capacités de la machine en terme de processeur et de mémoire vive.



• Une fois la machine initialisée, nous pouvons l’allumer pour la première fois. Après quelques secondes de chargement, l’assistant d’installation s’ouvre. Celui-ci va commencer par nous interroger sur le choix de la langue et la disposition du clavier. Nous laissons les choix sur langue française et clavier français. Le prochain panneau de l’assistant est important, puisqu’il permet de choisir le type d’OS à installer. Les différentes versions de Windows Server 2012 seront listés (Datacenter, Foundation, Standard ou encore Essentials), avec le choix d’une interface graphique ou en ligne de commandes pour chacuns d’eux. Dans le cas présent, nous indiquons que l’installation concerne un Windows Server 2012 RS Standard avec interface graphique.



• La prochaine étape consiste à choisir le type d’installation et l’emplacement de cette dernière. Nous indiquons que l’installation sera personnalisé, et nous sélectionnons l’unique disque virtuel disponible.



• L’installation va ensuite débuter d’elle même et prendre un certain temps avec de se terminer. La machine est susceptible de redémarrer pendant l’installation. Une fois l’installation effectuée à 100%, la machine va se lancer d’elle même, et nous proposer de définir les options d’authentification pour le compte administrateur de la machine.



• La machine va redémarrer une dernière fois avant de nous laisser le soin de s’authentifier pour la première fois sur le système, avec le mot de passe définit quelques instants auparavant.

III - Configuration de base.

• La machine est maintenant officiellement installée, et nous avons accès au bureau, au tableau de bord du serveur, et à toutes les fonctionnalités possibles. Avant de commencer à installer des services spécifiques, il reste deux étapes cruciales : la configuration réseau et la configuration du nom netbios. La configuration réseau s’effectue au sein du centre réseau et partage, dans les propriétés de la carte réseau, sous l’onglet Protocole Internet version 4 (TCP/IPv4). Etant donné que le serveur risque d’avoir une adresse IP fixe, c’est le moment de l’indiquer.



• Enfin, le nom netbios doit être modifié. Celui-ci est une suite de caractères alphanumériques aléatoire par défaut, ce qui risque de causer des conflits avec certains services. Pour le changer, rendons-nous sur le tableau de bord, dans l’onglet “Serveur local”, en face de la ligne “Nom de l’ordinateur”. En cliquant sur le nom actuel de la machine, une fenêtre nous proposera de modifier le nom netbios.



• Pour que les paramètres netbios soient à jour, un redémarrage sera nécessaire pour continuer. Pour terminer, si le serveur a besoin d'accueillir des logiciels tiers provenant d’internet, ou qu’il est nécessaire de naviguer sur Internet Explorer pour une raison ou une autre, il est plus commode de désactiver temporairement les protections de ce dernier. Pour ce faire, rendons-nous encore une fois dans les paramètres du serveur local sur le tableau de bord. La ligne à identifier se nomme “Configuration de sécurité renforcée d’internet explorer”. Par défaut, celle-ci est paramétré sur “Activé”. Il suffit de la passer en mode “inactif”.



• Il reste enfin à ouvrir internet explorer, dans l’onglet des paramètres, sous “Sécurité”, et d’abaisser le niveau de sécurité pour internet.



• Attention tout de même aux conséquences de tels modifications sur un serveur en production.
Et voilà, le serveur est maintenant prêt à accueillir différents services !
